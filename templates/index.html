What is WebSub?

WebSub is a W3C standard that describes a way in which services can "subscribe" to pages on your site and receive notifications when you publish new content.

Using WebSub means that compatible feed readers can show your website content to those who have subscribed to your site almost immediately after you have published your content.

How does WebSub work?

There are a few roles you will need to know before learning how WebSub works:

  • Hub: The server that sends notifications when content has been posted.
  • Client: The tool that subscribes to the server.
  • Website: The site that is publishing content (example: a blog)

Here are the steps that must be completed for a client (example: a feed reader) to subscribe to a website (example: a blog):

  1. The client discovers your WebSub endpoint that you list on your website.
  2. The client sends a request to "subscribe" to a page on your site (example: your home page).
  3. The hub will verify that the request was made by the client.
  4. Upon verification, the hub will send "fat pings" every time new content is posted on a page to which there is a subscriber. A fat ping is a notification that contains the contents of the new (or updated) web page.

Consuming clients must have a unique endpoint on their site to which they receive WebSub requests from each channel. This is described in more detail in the resources linked below.

Using WebSub, the hub will send notifications to the client when your website posts new content, in real-time.

How can I learn more about WebSub?

The IndieWeb wiki contains information on WebSub. You will find information on the basic principles behind WebSub, how to implement WebSub, and more.

Some useful resources to look at are the following pages:

Can I host my own WebSub server?

Yes! You can host your own WebSub endpoint. This client and server is open-sourced so you can use this one if you would like. You can deploy a version of this Micropub client and server by going to the project GitHub page and following the instructions.

You can find other implementation examples on the IndieWeb wiki.

This WebSub Server is not open for general use. You will only be able to use this service if you deploy it on your own infrastructure.
